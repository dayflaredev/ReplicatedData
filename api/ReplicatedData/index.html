ReplicatedData

ReplicatedData is a server-authoritative key/value store that automatically replicates data from the server to clients.

Types​

Scope​

</>
type Scope = string

A string identifier that groups related keys together. For example, "player_123" or "global".

Key​

</>
type Key = string

A string identifier for a value within a Scope.

Value​

</>
type Value = any

Any value that can be stored and replicated.

ChangeCallback​

</>
type ChangeCallback = (
newValue: Value,
oldValue: Value,
key: Key,
scope: Scope
) → ()

Callback fired when a value changes. Receives the new value, the old value, and the key/scope that changed.

Middleware​

</>
type Middleware = (
key: Key,
scope: Scope,
newValue: Value,
oldValue: Value
) → (
boolean,
Value?
)

A function that intercepts writes before they are applied.

Return false to block the write entirely. Return true, transformedValue to allow the write with an optionally transformed value. Return true (no second return) to allow the write unchanged.

Functions​

Get​

</>
ReplicatedData:Get(
key: Key,
scope: Scope
) → Value

Returns the value stored at key within scope, or nil if not set.

WaitFor​

This is a yielding function. When called, it will pause the Lua thread that called the function until a result is ready to be returned, without interrupting other scripts. Yields
</>
ReplicatedData:WaitFor(
key: Key,
scope: Scope
) → Value

Yields until key within scope is non-nil, then returns the value. Polls every 0.1 seconds.

GetScope​

</>
ReplicatedData:GetScope(scope: Scope) → {[Key]: Value}

Returns a shallow copy of all key/value pairs in scope. Returns an empty table if the scope does not exist.

GetAll​

</>
ReplicatedData:GetAll() → {[Scope]: {[Key]: Value}}

Returns a shallow copy of the entire store across all scopes.

Has​

</>
ReplicatedData:Has(
key: Key,
scope: Scope
) → boolean

Returns true if key exists in scope, even when its value is false or 0.

Keys​

</>
ReplicatedData:Keys(scope: Scope) → {Key}

Returns an array of all keys present in scope.

Scopes​

</>
ReplicatedData:Scopes() → {Scope}

Returns an array of all known scope names.

Set​

This item only works when running on the server. Server
</>
ReplicatedData:Set(
key: Key,
scope: Scope,
value: Value
) → boolean

Sets key in scope to value. Runs the middleware stack before applying. Returns true if the value changed, false if it was blocked or unchanged.

Server only

Writes should only originate on the server. The server will automatically replicate the change to all eligible clients.

Update​

This item only works when running on the server. Server
</>
ReplicatedData:Update(
key: Key,
scope: Scope,
updater: (current: Value) → Value
) → boolean

Applies updater to the current value of key in scope and sets the result. updater receives the current value (or nil) and should return the new value.

Increment​

This item only works when running on the server. Server
</>
ReplicatedData:Increment(
key: Key,
scope: Scope,
amount: number?--

Defaults to 1

) → boolean

Increments a numeric key by amount (default 1). If the key is not yet set, it starts from 0.

Delete​

This item only works when running on the server. Server
</>
ReplicatedData:Delete(
key: Key,
scope: Scope
) → ()

Deletes key from scope. Fires listeners with nil as the new value.

DeleteScope​

This item only works when running on the server. Server
</>
ReplicatedData:DeleteScope(scope: Scope) → ()

Deletes every key in scope, then removes the scope itself.

BeginBatch​

This item only works when running on the server. Server
</>
ReplicatedData:BeginBatch() → ()

Begins a batch operation. All subsequent ReplicatedData.BatchSet calls will be queued rather than applied immediately. Call ReplicatedData.CommitBatch to apply or ReplicatedData.RollbackBatch to discard.

BatchSet​

This item only works when running on the server. Server
</>
ReplicatedData:BatchSet(
key: Key,
scope: Scope,
value: Value
) → ()

Queues a write when inside a batch (after ReplicatedData.BeginBatch). Falls through to ReplicatedData.Set immediately if no batch is active.

CommitBatch​

This item only works when running on the server. Server
</>
ReplicatedData:CommitBatch() → number

Applies all queued batch writes and ends the batch. Returns the number of keys that actually changed.

RollbackBatch​

This item only works when running on the server. Server
</>
ReplicatedData:RollbackBatch() → ()

Discards all queued batch writes and ends the batch without applying anything.

UseMiddleware​

</>
ReplicatedData:UseMiddleware(handler: Middleware) → ()

Registers a Middleware handler. Handlers are executed in insertion order on every ReplicatedData.Set and ReplicatedData.Update call.

Middleware does not run on internal _apply* bridge calls.

RemoveMiddleware​

</>
ReplicatedData:RemoveMiddleware(handler: Middleware) → ()

Removes a previously registered Middleware handler.

OnChange​

</>
ReplicatedData:OnChange(
key: Key,
scope: Scope,
callback: ChangeCallback
) → () → ()

Subscribes to changes on a specific key within scope. Returns a function that disconnects the listener when called.

OnScopeChange​

</>
ReplicatedData:OnScopeChange(
scope: Scope,
callback: ChangeCallback
) → () → ()

Subscribes to changes on any key within scope. Returns a function that disconnects the listener when called.

OnAnyChange​

</>
ReplicatedData:OnAnyChange(callback: ChangeCallback) → () → ()

Subscribes to every change in the entire store, regardless of scope or key. Returns a function that disconnects the listener when called.

Observe​

</>
ReplicatedData:Observe(
key: Key,
scope: Scope,
callback: ChangeCallback
) → () → ()

Subscribes to key in scope and immediately fires callback with the current value. Useful for initialising UI without separate init logic.

Once​

</>
ReplicatedData:Once(
key: Key,
scope: Scope,
callback: ChangeCallback
) → () → ()--

Disconnect function (if you need to cancel before it fires)

Subscribes to key in scope and automatically disconnects after the first change fires.

SetScopeFilter​

This item only works when running on the server. Server
</>
ReplicatedData:SetScopeFilter(
scope: Scope,
fn: (
player: Player,
key: Key,
value: Value
) → boolean
) → ()

Sets a filter function for scope that controls which players receive its data.

The filter runs on every write and on the initial sync for joining players. Return true to allow replication to that player, false to hide it.

RemoveScopeFilter​

This item only works when running on the server. Server
</>
ReplicatedData:RemoveScopeFilter(scope: Scope) → ()

Removes the replication filter for scope, allowing all players to receive it.

_ensureServerSync​

This item only works when running on the server. Server
</>
ReplicatedData:_ensureServerSync() → ()

Initialises server-side replication. Must be called once from a Script.

  • Listens for store changes and pushes deltas to eligible clients.
  • Sends a filtered full snapshot to each player when they join.

_ensureClientSync​

This item only works when running on the client. Client
</>
ReplicatedData:_ensureClientSync() → ()

Initialises client-side replication. Must be called once from a LocalScript.

  • Applies the full snapshot received on join.
  • Listens for incremental set/delete packets from the server.
